### Projeto SacBase ###

1 - Ferramentas de desenvolvimento
  1.1 - Front End
    - HTML
    - CSS
    - JavaScript Vanila
    - Bootstrap 5

  1.2 - Back End
    - Python
    - Django Framework

  1.3 - Banco de Dados
    - Postgresql

2 - Estrutura das pastas e arquivos principais
  - SAC_BASE # pasta raiz do projeto
    - sac_base # pasta do aplicativo principal
      - context_processors.py # Mantém sempre estrutura do contrato da sisVar nas respostas e a alimenta com os dados do BackEnd por meio de request.sisvar_extra
      - form_validador.py # Primeira camada de validação dos formuários no BackEnd. Utiliza o schema do formulário para validar.
    - pages # pasta onde ficam os aplicativos
      - usuario # pasta do aplicativo para gestão de usuáios - login, logout, alterar senha, cadastro de usuário, permissões
        - middleware.py # Verifica se o usuário está autenticado
    - static # pasta estática global
      - css
       - styles.css # estilos globais
      - js
        - base.js # JavaScript global, importado por base.html
        - refresh_varSis.js # Responsável por hidratar sisVar com os dados dos formulários
        - sisVar.js # Variável de estado global e fonte única dos dados
        - input_rules.js # Responsável por regras de validação e máscaras dos inputs
        - loader.js # Responsável por gerenciar o loader global da aplicação, um spinner que indica que algo está carregando
    - base.html

3 - Lógica do sistema
  3.1 - sisVar: Variável de estado global e fonte única dos dados;
    - Todos os formulários consomem e alimentam a sisVar;
    - Só a sisVar é enviada para o servidor, total ou parcialmente;

    3.2 - Resposta/Tipo de requisição
        Requisições GET: return render(request, template)

        Requisições POST: return JsonResponse()
